Funciones

Las funciones son esenciales para organizar y estructurar tu código, haciéndolo más modular y fácil de entender.

  • Funciones tradicionales:Se declaran utilizando la palabra clave function, seguida de un nombre (opcional si es anónima), una lista de parámetros entre paréntesis (también opcional), y un bloque de código entre llaves {}
    • 
            // Función con nombre
                function saludar(nombre) {
                console.log("Hola, " + nombre + "!");
                }

                saludar("Carlos"); // Llama a la función

                // Función anónima (asignada a una variable)
                const multiplicar = function(a, b) {
                return a * b;
                };

                let resultado = multiplicar(5, 4);
                console.log(resultado); // Imprime 20
  • Arrow functions (=>): Introducidas en ES6 (ECMAScript 2015), proporcionan una sintaxis más concisa para escribir funciones anónimas. No tienen su propio this (heredan el this del contexto léxico circundante), lo que las hace especialmente útiles en ciertos escenarios.
    • 
            // Sintaxis básica
            const sumar = (a, b) => {
            return a + b;
            };

            // Si solo hay un parámetro, los paréntesis son opcionales
            const cuadrado = numero => {
            return numero * numero;
            };

            // Si la función solo retorna un valor, las llaves y la palabra clave 'return' son opcionales
            const cubo = numero => numero ** 3;

            console.log(sumar(2, 3));    // Imprime 5
            console.log(cuadrado(5)); // Imprime 25
            console.log(cubo(3));     // Imprime 27
  • Funciones callback:Una función callback es una función que se pasa como argumento a otra función. La función que recibe el callback puede ejecutar esta función en algún momento posterior. Son fundamentales en operaciones asíncronas y en la programación basada en eventos.
    • 
           function procesar(valor, callback) {
            console.log("Procesando valor: " + valor);
            callback(valor * 2); // Llama a la función callback con el valor procesado
            }

            function mostrarResultado(resultado) {
            console.log("El resultado es: " + resultado);
            }

            procesar(5, mostrarResultado); // 'mostrarResultado' es el callback
  • Funciones recursivas:Una función recursiva es una función que se llama a sí misma dentro de su propio cuerpo. Es importante tener una condición de parada (caso base) para evitar una recursión infinita y un error de desbordamiento de pila (stack overflow)
    • 
            function factorial(n) {
                if (n === 0) { // Caso base
                    return 1;
                } else {
                    return n * factorial(n - 1); // Llamada recursiva
                }
                    }

                console.log(factorial(5)); // Imprime 120 (5 * 4 * 3 * 2 * 1)
  • Funciones autoejecutables :Son funciones que se definen y se ejecutan inmediatamente después de su creación. Se utilizan a menudo para crear un scope privado y evitar la contaminación del scope global.
    • 
            // Sintaxis común usando paréntesis alrededor de la definición de la función
            (function() {
            let mensajeInterno = "Hola desde la IIFE!";
            console.log(mensajeInterno);
            })();

            // Otra sintaxis común
            (() => {
            const otroMensaje = "Otra IIFE con arrow function.";
            console.log(otroMensaje);
            })();
